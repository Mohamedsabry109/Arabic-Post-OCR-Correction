# End-to-End Smoke Test — Test Samples (250)

> **NOTE (v1.0.0)**: Phase numbering changed. Phases 4A (Rules) and 4B (QALB Few-Shot) were removed.
> Phase 4C → Phase 5 (CAMeL Validation). Phase 4D → Phase 4 (Self-Reflective). 
> Old Phase 5/6 Combinations → Phase 6. The steps below reference the old phase names — update
> commands to use `run_phase4.py` (Phase 4), `run_phase5.py` (Phase 5 CAMeL + Phase 6 combos).

Tests all 7 phases + automatic prompt engineering using 250 test samples
selected across all PATS-A01 train fonts + KHATT-train. Samples are stratified
by CER difficulty (correct / easy / medium / hard) with font diversity and
randomization.

**Sample list**: `data/test_samples.json` (generated by `scripts/select_test_samples.py`)
**Datasets**: 9 datasets (8 PATS-A01 fonts + KHATT-train), auto-resolved from sample list

---

## How the workflow works

The local machine has no GPU. All LLM inference runs on Kaggle (or Colab).
Each phase that needs inference follows this pattern:

```
LOCAL: export  -->  produces results/<phase>/inference_input.jsonl
LOCAL: upload  -->  upload JSONL to a Kaggle dataset (via Kaggle UI or API)
LOCAL: git push --> push latest code to GitHub (Kaggle clones the repo)
KAGGLE: clone  -->  git clone the repo inside the notebook
KAGGLE: infer  -->  python scripts/infer.py --input ... --output ...
KAGGLE: save   -->  download corrections.jsonl from Output tab (or HF sync)
LOCAL: copy    -->  place corrections.jsonl into results/<phase>/
LOCAL: analyze -->  compute metrics against ground truth
```

**Key detail**: `results/` is gitignored, so JSONL files cannot travel via
git push. They must be uploaded to Kaggle as a dataset and downloaded back
manually (or synced via HuggingFace — see `docs/Kaggle_Colab_Guide.md`).

## Prerequisites

### 1. Generate test samples (LOCAL)

```bash
# Default: 250 samples (25 correct + 25 easy + 100 medium + 100 hard)
python scripts/select_test_samples.py --include-khatt

# Or use --n to auto-distribute (10% correct, 10% easy, 40% medium, 40% hard)
python scripts/select_test_samples.py --n 200 --include-khatt

# Or set each bucket size explicitly
python scripts/select_test_samples.py --n-correct 20 --n-easy 30 --n-medium 100 --n-hard 100 --include-khatt

# Custom seed for different random selection
python scripts/select_test_samples.py --seed 123 --include-khatt
```

### 2. Verify samples (LOCAL)

```bash
python -c "
import json, yaml, sys; sys.path.insert(0, '.')
from src.data.data_loader import DataLoader
with open('configs/config.yaml', encoding='utf-8') as f: config = yaml.safe_load(f)
with open('data/test_samples.json', encoding='utf-8') as f: data = json.load(f)
sample_ids = set(data['sample_ids']); loader = DataLoader(config)
count = sum(1 for ds in data['meta']['by_dataset'] for s in loader.iter_samples(ds, sample_ids=sample_ids))
bc = data['meta']['by_category']
print(f'{count}/{data[\"meta\"][\"total_selected\"]} samples loaded OK')
print(f'  Correct: {bc[\"correct\"]}, Easy: {bc[\"easy\"]}, Medium: {bc[\"medium\"]}, Hard: {bc[\"hard\"]}')
"
```

### 3. Kaggle setup (one-time)

1. Create a Kaggle notebook with **GPU T4 x2** and **Internet enabled**.
2. Add your GitHub repo URL (or token for private repos).
3. Optionally set up HF sync: create a HuggingFace dataset repo, add
   `HF_TOKEN` as a Kaggle secret (Add-ons > Secrets).
4. See `notebooks/kaggle_setup.ipynb` for a ready-to-use template.

## Automatic Prompt Engineering (`scripts/craft_prompt.py`)

Uses the LLM itself to design an optimal correction prompt, grounded in real
OCR error patterns. This is independent of the 6-phase pipeline and produces a
`crafted_system_prompt.txt` that can be evaluated against Phase 2 baselines.

**Output directory**: `results/prompt_craft/`

### Round 1 — Generate the crafted prompt

#### Step 1: Prepare meta-prompt (LOCAL)

Selects stratified samples (correct / easy / medium / hard) from train data
and builds a meta-prompt asking the LLM to design a correction system prompt.

```bash
python scripts/craft_prompt.py --mode prepare --include-khatt
```

**Produces**:
- `results/prompt_craft/meta_prompt_input.jsonl` (1 record — the meta-prompt)
- `results/prompt_craft/samples_used.json` (which samples were picked)
- `results/prompt_craft/meta_prompt.txt` (human-readable copy)

#### Step 2: Upload + Push (LOCAL)

```bash
# Push latest code to GitHub
git add -A && git commit -m "prompt craft prepare" && git push

# Upload meta_prompt_input.jsonl to your Kaggle dataset
```

#### Step 3: Inference (KAGGLE)

In your Kaggle notebook (clone repo first if new session):

```bash
python /kaggle/working/project/scripts/infer.py \
    --input  /kaggle/input/YOUR_DATASET_NAME/meta_prompt_input.jsonl \
    --output /kaggle/working/meta_prompt_output.jsonl \
    --model  Qwen/Qwen3-4B-Instruct-2507
```

This is a single inference call — the LLM reads the error examples and
generates an Arabic system prompt.

#### Step 4: Download (KAGGLE -> LOCAL)

Download `meta_prompt_output.jsonl` from the Kaggle Output tab and copy it:

```bash
cp ~/Downloads/meta_prompt_output.jsonl results/prompt_craft/meta_prompt_output.jsonl
```

#### Step 5: Extract crafted prompt (LOCAL)

```bash
python scripts/craft_prompt.py --mode extract
```

**Produces**: `results/prompt_craft/crafted_system_prompt.txt`

Review the generated Arabic prompt before proceeding to Round 2.

### Round 3 (optional) — Refine based on failures

Only needed if the crafted prompt caused regressions (samples where LLM
output was worse than the original OCR).

#### Step 1: Build refinement prompt (LOCAL)

Reads the crafted prompt + the evaluation corrections, identifies regressions,
and builds a new meta-prompt showing the failures to the LLM.

```bash
python scripts/craft_prompt.py --mode refine
```

**Produces**:
- `results/prompt_craft/refine_input.jsonl` (1 record — the refine prompt)
- `results/prompt_craft/regressions.json` (all regression details)
- `results/prompt_craft/refine_prompt.txt` (human-readable copy)

If no regressions are found, the script reports success and stops — no
refinement needed.

#### Step 2: Upload + Push (LOCAL)

Upload `results/prompt_craft/refine_input.jsonl` to your Kaggle dataset.

#### Step 3: Inference (KAGGLE)

```bash
python /kaggle/working/project/scripts/infer.py \
    --input  /kaggle/input/YOUR_DATASET_NAME/refine_input.jsonl \
    --output /kaggle/working/refine_output.jsonl \
    --model  Qwen/Qwen3-4B-Instruct-2507
```

#### Step 4: Download + copy (KAGGLE -> LOCAL)

```bash
cp ~/Downloads/refine_output.jsonl results/prompt_craft/refine_output.jsonl
```

#### Step 5: Extract revised prompt (LOCAL)

```bash
python scripts/craft_prompt.py --mode extract \
    --corrections results/prompt_craft/refine_output.jsonl
```

This overwrites `crafted_system_prompt.txt` with the revised version.

#### Step 6: Repeat Round 2

Run Round 2 again (export -> upload -> Kaggle infer -> download -> analyze) to
evaluate the revised prompt. Compare metrics to see if regressions decreased.
