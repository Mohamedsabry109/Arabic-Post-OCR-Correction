\section*{C.4 \quad Development Config (\texttt{config\_dev.yaml})}
%----------------------------------------------------------------------------

A lightweight development configuration is provided for local pipeline
validation without a GPU. It restricts evaluation to two datasets (Akhbar and
KHATT), limits to 20 samples per dataset, and switches the backend to
\texttt{mock} (which returns OCR text unchanged, producing CER equal to the
OCR baseline).

\begin{lstlisting}
_base_: "config.yaml"

model:
  backend: "mock"

datasets:
  - name: "PATS-A01-Akhbar-val"
    font: "Akhbar"
    type: "PATS-A01"
    pats_split: "validation"
  - name: "KHATT-validation"
    split: "validation"
    type: "KHATT"

dev:
  limit: 20
\end{lstlisting}

\noindent
With this config a full export--infer--analyze cycle completes on the local
machine in under one minute, producing valid JSON outputs that can be used to
validate pipeline logic without requiring cloud GPU access.
