\title{Does More Context Help? Evaluating Knowledge-Augmented Prompt Strategies
for Arabic Post-OCR Correction with a Lightweight LLM}

\maketitle

\begin{abstract}
Arabic optical character recognition (OCR) systems often produce systematic
character-level errors---dot confusions, hamza substitutions, and taa marbuta
swaps---that degrade downstream text processing. Post-OCR correction using
large language models (LLMs) is a natural remedy, but it is unclear how much
additional context actually helps. This paper reports a controlled experiment
in which six prompt-engineering strategies are evaluated on the PATS-A01
dataset using Qwen3-4B-Instruct-2507 as the correction model. The strategies
tested are: zero-shot correction, confusion-matrix injection, rule-augmented
prompting, few-shot examples from the QALB corpus, self-reflective prompting,
and retrieval-augmented generation with the OpenITI corpus. Against expectations,
zero-shot correction achieves the lowest character error rate (CER) among all
LLM-based conditions at 5.77\%, compared to a raw OCR CER of 17.01\% driven
by a Qaari repetition artifact. Every augmentation strategy degrades performance
relative to zero-shot, with RAG being the worst offender (CER~22.10\%) and
few-shot examples performing best among the augmented conditions (CER~10.09\%).
Morphological post-processing via CAMeL Tools neither helps nor hurts. These
results suggest that for high-quality typewritten Arabic text, prompt enrichment
causes over-correction rather than targeted repair, and that the most reliable
LLM strategy is also the simplest one.
\end{abstract}

\begin{IEEEkeywords}
Arabic OCR, post-OCR correction, large language models, prompt engineering,
retrieval-augmented generation, Qwen3, PATS-A01

\section{Introduction}
% -------------------------------------------------------------------------

Arabic optical character recognition is a harder problem than it looks. The
script is cursive, dots are phonemically significant, and a single misplaced
dot can change the meaning of a word entirely. Open-source OCR engines such
as Qaari~\cite{qaari} have made Arabic recognition more accessible, but they
still produce character-level errors that compound in downstream tasks like
information retrieval or machine translation.

Post-OCR correction---treating the OCR output as noisy text and using a language
model to clean it---has seen renewed interest since powerful instruction-tuned
LLMs became widely available. The appeal is obvious: no training data is needed,
the correction can be done in a single forward pass, and the model's language
priors should naturally prefer valid Arabic words. What is less obvious is
whether feeding the model more context actually improves the result.

We set up a controlled experiment to test exactly this. Starting from zero-shot
correction as a baseline, we progressively enriched the prompt with different
knowledge sources: the OCR engine's known confusion patterns, explicit Arabic
spelling rules, error--correction examples from a corpus, the model's own
diagnosed failure modes from training-split analysis, and retrieved passages
from a large Arabic corpus. Six conditions in total, all evaluated on the same
dataset and model, so the only variable is the prompt.

The main finding is negative in a useful way. None of the enrichments beat the
zero-shot baseline; most made things noticeably worse. That outcome is informative
on its own: it says something about how instruction-tuned LLMs process extra context
and, specifically, how they tend to treat it as a license to rewrite rather than a
guide to be careful.

The contributions of this work are:
\begin{itemize}
\item A systematic comparison of six prompt strategies for Arabic post-OCR
      correction under controlled conditions.
\item Evidence that zero-shot correction outperforms all knowledge-augmented
      variants on high-quality typewritten Arabic OCR output.
\item A characterization of the Qaari repetition artifact and how LLM correction
      interacts with it.
\item An analysis of why each augmentation strategy fails, with implications for
      future system design.
\end{itemize}

\section{Related Work}
% -------------------------------------------------------------------------

\subsection{Arabic OCR and Its Error Patterns}
Arabic OCR has been studied since the 1990s \cite{arabic_ocr_survey}, and the
core difficulties are well documented: the cursive baseline makes segmentation
hard, diacritics are often omitted or misread, and several letter groups (ba/ta/tha,
dal/dhal, ra/zay) share the same skeleton differing only in dots. Qaari, the
open-source engine used in this study, is based on Tesseract with Arabic-specific
training and inherits both its strengths and its failure modes \cite{qaari}.

\subsection{Post-OCR Correction}
Traditional post-OCR correction relied on language models, lexicon lookup, or
a combination of both \cite{afli2016,dong2018}. The move toward neural approaches
brought sequence-to-sequence models that treat correction as a translation
task \cite{nguyen2021}. More recently, prompted LLMs have been tested for
English document correction with encouraging results \cite{llm_ocr_correction},
but Arabic-specific studies remain sparse. The most directly relevant prior work
uses Arabic BERT variants for post-hoc correction \cite{arabic_bert_ocr}, but
the prompt-engineering perspective has not been explored systematically.

\subsection{Knowledge-Augmented Prompting}
Injection of structured knowledge into LLM prompts has been shown to help in
several settings: few-shot examples improve performance across tasks
\cite{brown2020}, chain-of-thought prompts help with reasoning \cite{wei2022},
and retrieval-augmented generation grounds outputs in factual content
\cite{lewis2020}. There is also work on using a model's own error statistics
to guide its future behavior, sometimes called self-reflection or self-critique
\cite{shinn2023}. Our study applies these ideas to OCR correction and
measures whether the gains translate.

\subsection{Arabic NLP Resources}
The QALB corpus provides Arabic text with corresponding corrections for human
typing errors \cite{mohit2014}, making it the closest available resource to
OCR correction examples. OpenITI is a large digitized corpus of classical and
modern Arabic texts \cite{romanov2020}, suitable for retrieval-based grounding.
CAMeL Tools provides morphological analysis and disambiguation
for modern standard Arabic \cite{obeid2020}, which we use for post-correction
validation.

% -------------------------------------------------------------------------
\section{Dataset and Experimental Setup}
% -------------------------------------------------------------------------

\subsection{Dataset}
All experiments in this paper use the PATS-A01 dataset \cite{pats}, specifically
the Akhbar font subset. PATS-A01 is a benchmark for Arabic document recognition
covering eight typewritten fonts on synthetic document images. The Akhbar font
represents a common Arabic newspaper typeface and is representative of the
typewritten subset. We evaluate on 200 samples from the training split,
using the corresponding ground-truth text files for CER and WER
computation.\footnote{Diacritics are stripped before evaluation to avoid
penalizing the model for diacritic insertion or deletion, which is not the
target of this correction task.}

\subsection{OCR Engine}
We use Qaari as the OCR engine. Qaari's output serves as both the baseline
for Phase~1 analysis and the input to all LLM correction phases.

\subsection{Correction Model}
Qwen3-4B-Instruct-2507 \cite{qwen3} is used as the LLM for all correction
phases. It is a 4-billion-parameter instruction-tuned model with strong
multilingual capabilities and native Arabic support. Inference was run on
a GPU-accelerated environment (Kaggle). The \texttt{enable\_thinking} flag
was set to \texttt{False} to suppress the chain-of-thought scratchpad, which
is not needed for this task and increases token cost without measurable benefit.

\subsection{Evaluation Metrics}
Character Error Rate (CER) is the primary metric:
\begin{equation}
\text{CER} = \frac{S + D + I}{N},
\label{eq:cer}
\end{equation}
where $S$, $D$, $I$ are the number of substitutions, deletions, and insertions
at the character level, and $N$ is the total number of characters in the
reference. Word Error Rate (WER) is reported alongside CER for completeness.
Lower values indicate better performance.

\subsection{Pipeline Architecture}
Each experimental phase follows a three-stage pipeline: (1)~\textit{export},
which prepares the inference input JSONL from the local dataset; (2)~\textit{infer},
which runs the LLM on a GPU-capable environment; and (3)~\textit{analyze},
which computes metrics and comparison statistics locally. This separation
means the correction model itself never needs to run where the ground-truth
data is stored, and it allows easy replication on different hardware.

\section{Experimental Methodology}
% -------------------------------------------------------------------------

We organize the experiment into six phases. Phase~1 characterizes the OCR
errors. Phase~2 is the zero-shot LLM correction baseline. Phases~3 through~5
(including 4A--4D) each introduce exactly one form of additional knowledge.
The design ensures that each augmentation can be attributed to a single
variable change relative to Phase~2.

\subsection{Phase 1: OCR Baseline and Error Taxonomy}

Before correcting anything, we need to understand what there is to correct.
Phase~1 runs Qaari over the 200 samples and computes CER and WER against
ground truth. It also builds a character-level confusion matrix and classifies
errors into seven categories: dot confusion (ba/ta/tha/ya misreads), hamza
errors, taa marbuta/ha swaps, alef-maksura/ya swaps, similar-shape pairs
(ra/zay, dal/dhal), merged words, split words, and other substitutions.

Morphological validity of OCR tokens is checked with CAMeL Tools. Words that
fail morphological analysis are classed as non-word errors; those that parse
successfully but differ from the ground truth are valid-but-wrong errors. This
distinction matters because non-word errors should be straightforward for an LLM
to catch (the output is not a real Arabic word), whereas valid-but-wrong errors
require semantic understanding.

\subsection{Phase 2: Zero-Shot LLM Correction (Baseline)}

The zero-shot prompt instructs the model to correct OCR errors in the supplied
text and return only the corrected output without explanation. The system prompt
states the task in Arabic and specifies that if the text appears correct, it
should be returned unchanged. This conservative formulation (``only fix clear
errors'') was chosen to reduce unnecessary rewrites, which are a known failure
mode for instruction-following models on text that is mostly correct.

Phase~2 output serves as the comparison point for all subsequent phases. A
phase is considered beneficial only if it achieves lower CER than Phase~2.

\subsection{Phase 3: Confusion-Matrix Injection}

Phase~3 extends the system prompt with a list of Qaari's most frequent
character confusions extracted from the Phase~1 confusion matrix. Specifically,
the top-10 character-pair confusions are formatted as a numbered list in Arabic
and injected before the correction instruction. The hypothesis is that telling
the model which characters Qaari tends to mix up should focus its attention on
those specific pairs.

\subsection{Phase 4A: Rule-Augmented Prompting}

Arabic orthography has several rule-governed phenomena that OCR engines
frequently mishandle: the distinction between hamza al-qat' and hamza al-wasl,
the taa marbuta vs.\ terminal ha, the alef maqsura vs.\ ya, and the moon/sun
letter assimilation in the definite article. Phase~4A injects a curated set of
these rules as numbered instructions before the correction directive.

\subsection{Phase 4B: Few-Shot Examples from QALB}

The QALB corpus provides pairs of erroneous Arabic sentences and their
corrections, originally collected for human grammatical error correction.
Phase~4B selects five diverse pairs from the OCR-relevant subset of QALB
(filtered to pairs where errors overlap with known OCR error types) and formats
them as input--output demonstrations in the prompt. The selection strategy
prioritizes error-type diversity rather than random sampling.

\subsection{Phase 4C: CAMeL Morphological Post-Processing}

Unlike the other phases, Phase~4C does not modify the prompt at all. Instead,
it applies a post-processing step after zero-shot correction: each word in the
LLM output is checked against the CAMeL Tools morphological analyzer. If a
corrected word fails morphological analysis \textit{and} the original OCR word
passes it, the corrected word is reverted to the OCR original. The logic is
conservative by design---it only intervenes when the LLM has produced a
morphologically invalid word from a valid source, suggesting an unwarranted
rewrite.

\subsection{Phase 4D: Self-Reflective Prompting}

Phase~4D feeds the model a statistical summary of its own past mistakes. To
generate this summary, Phase~2 is run on the training split (where ground truth
is available), and the corrections are compared to ground truth by an error
analyzer that tracks the fix rate and error introduction rate per error type.
The results reveal, for instance, that the model fixes insertion errors 97\%
of the time but introduces new deletion errors at a high rate. These statistics
are reformatted as Arabic-language diagnostic notes---``you tend to over-correct
X; be cautious with Y''---and injected into the system prompt for the
validation-split run. This constitutes a lightweight form of meta-learning
without any fine-tuning.

\subsection{Phase 5: Retrieval-Augmented Generation with OpenITI}

Phase~5 uses the OpenITI Arabic corpus as an external knowledge source. A FAISS
vector index is built over 200,000 corpus sentences using multilingual sentence
embeddings (paraphrase-multilingual-MiniLM-L12-v2). For each OCR input, the
three most similar sentences are retrieved and included in the prompt as reference
text. The hypothesis is that seeing syntactically and semantically similar correct
Arabic text helps the model produce more faithful corrections.

\section{Results}
% -------------------------------------------------------------------------

\subsection{Phase 1: How Bad Is Qaari on This Dataset?}

The raw OCR CER across all 200 samples is 17.01\% (WER: 20.14\%).
This number, however, is dominated by a single pathological sample in which
Qaari enters a repetition loop, generating the same short phrase hundreds of
times instead of the actual document text. When this sample is excluded, the CER
drops to 1.37\% (WER: 5.04\%), revealing that Qaari's character-level
accuracy on normal Akhbar-font input is actually quite high.

The confusion matrix shows that the dominant error type---by far---is
\textit{insertion}, accounting for 1,964 of 2,063 total character errors
(95.2\%). Most of these insertions come from the repetition sample. The
remaining errors split across dot confusion (0.68\%), other substitutions
(2.38\%), deletions (1.55\%), alef-maksura errors (0.15\%), and similar-shape
confusions (0.05\%).

The top character confusions on normal samples are: \texttt{T} (ta marbuta)
$\to$ \texttt{Z} (zal), ya $\leftrightarrow$ ba, alef-maksura $\to$ ya,
fa $\to$ qaf, and ta'a $\to$ tha'a. These align with the known dot-sensitivity
problem.

Morphological analysis with CAMeL Tools shows that the majority of OCR tokens
outside the repetition sample are morphologically valid Arabic words---the errors
are primarily of the valid-but-wrong type, not non-words. This is an important
observation for the phases that follow: the LLM cannot detect these errors by
lexical lookup alone; context is required.

\subsection{Phase 2: What Can Zero-Shot Do?}

Zero-shot LLM correction achieves CER~5.77\% and WER~10.60\%. Compared to the
all-samples OCR baseline (CER~17.01\%), this represents a reduction of 66\%
in CER. The improvement is largely attributable to the repetition sample: the
LLM recognizes the pathological repetition and collapses it into a coherent
text, dramatically cutting the error count for that sample.

On normal samples---the 199 samples without the repetition artifact---the
picture reverses. The OCR CER for normal samples is only 1.37\%, whereas the
LLM outputs on those same samples contribute to the overall 5.77\% figure.
This means the LLM introduces new errors into text that Qaari rendered correctly.
The over-correction hypothesis is already apparent at the zero-shot level.

The CER standard deviation is high (0.12), reflecting per-sample variance:
some samples benefit substantially while others are degraded. The median CER
(3.28\%) is lower than the mean, indicating that a small number of high-error
samples are pulling the average up.

These Phase~2 results become the reference point for all subsequent phases.
Any phase achieving CER below 5.77\% would represent a genuine improvement
over zero-shot.

No augmentation strategy improves over zero-shot. The degradation ranges from
mild (Phase~4B, $+75\%$ relative CER increase) to severe (Phase~5,
$+283\%$). The pattern is consistent across CER and WER.

\textbf{Phase 3 (Confusion Matrix).} CER rises from 5.77\% to 19.62\%, the
second worst result. Injecting the confusion list appears to activate the model's
correction instincts more aggressively on every token even when no confusion is
present, producing a net increase in substitutions. The median CER (1.67\%) is
much lower than the mean, suggesting that a subset of samples suffers catastrophic
rewrites while the majority fare similarly to zero-shot.

\textbf{Phase 4A (Rules).} CER rises to 11.03\%. The rule prompt pushes the model
to apply orthographic rules uniformly, but many of these rules govern contexts
that are not present in every sentence. The model generalizes from the rules
to adjacent patterns, making corrections that are technically rule-compliant but
contextually wrong. The effect is smaller than Phase~3 but still clearly negative.

\textbf{Phase 4B (Few-Shot).} At CER 10.09\%, few-shot is the best-performing
augmented condition, though still 75\% worse than zero-shot in relative terms.
The QALB examples are drawn from human typing errors; their error distribution
overlaps with OCR errors only partially. The model picks up the correction
behavior shown in the examples and applies it beyond the intended scope.

\textbf{Phase 4C (CAMeL Post-Processing).} CER is identical to Phase~2 at
5.77\%. Inspection of the outputs confirms that the revert strategy triggered
zero times: no word in the Phase~2 output was simultaneously morphologically
invalid and sourced from a valid OCR token. The CAMeL validation layer is
effectively inert on this data. This is partly by design---the conservative
revert strategy only activates in a specific crossing condition---but it also
reflects that Qaari's outputs on normal Akhbar samples are predominantly
morphologically valid to begin with.

\textbf{Phase 4D (Self-Reflective).} CER rises to 10.48\%. The self-diagnostic
notes, derived from training-split error analysis, tell the model that it over-corrects
insertions and struggles with deletions. These are accurate observations (the
Phase~4D insights show a 97.35\% fix rate for insertions but a 3.12\% fix rate
for deletions, with a 25\% error introduction rate overall). However, providing
this information in the prompt does not translate to behavioral change; instead,
the model seems to become more interventionist, introducing more errors than
zero-shot despite being told to be careful.

\textbf{Phase 5 (RAG).} With CER 22.10\%, RAG is the worst LLM condition. Retrieved
passages from OpenITI---classical and modern Arabic texts---are dissimilar in style
and vocabulary to the short newspaper sentences in PATS-A01. The model treats the
retrieved context as a template, pulling in vocabulary and constructions from the
retrieved sentences that do not belong in the input text. CER standard deviation is
also very high (1.99), indicating inconsistent behavior across samples.

\subsection{Summary}

The clearest takeaway from Table~\ref{tab:main} is that adding more information
to the prompt does not help and often hurts. Zero-shot is the ceiling, not the
floor. The model's built-in Arabic language knowledge is sufficient to catch
the Qaari repetition artifact; everything else it does is net harmful on
high-quality typewritten input.

% -------------------------------------------------------------------------
\section{Discussion}
% -------------------------------------------------------------------------

\subsection{Why Does Augmentation Backfire?}

The consistent degradation across very different augmentation types points to a
common underlying mechanism: the additional context raises the model's threshold
for leaving text unchanged. A zero-shot correction prompt with a conservative
instruction (``return as-is if correct'') keeps the model in a low-intervention
mode. Confusion lists, rules, examples, and diagnostic notes all signal that
something is wrong with the input and that active correction is expected. The
model obliges, rewriting portions of text that Qaari had actually got right.

This is a manifestation of what might be called \textit{prompt-induced
over-correction}. The phenomenon is not unique to Arabic OCR---similar effects
have been observed when chain-of-thought prompts cause models to second-guess
factual answers they would have got right under direct prompting. In the OCR
setting it is particularly costly because the baseline OCR is already accurate
on most tokens; any unnecessary rewrite creates an error.

\subsection{The Repetition Artifact}

The Qaari repetition artifact---where the engine loops over a short span of
characters---is an interesting edge case. Phase~2's 66\% CER improvement over
the all-samples OCR baseline is almost entirely attributable to the model
collapsing this single pathological sample. Without that sample, LLM correction
would appear to degrade OCR performance across the board. This highlights a
practical consideration: LLMs can serve as a safety net for extreme OCR failures
even when they harm ordinary outputs, and a sensible deployment might apply
LLM correction only to samples that exceed a CER threshold rather than
uniformly to all output.

\subsection{Why CAMeL Had No Effect}

The morphological revert strategy's inertness is informative. It tells us that
Phase~2's LLM errors are not primarily non-word insertions---they are valid Arabic
words that are wrong in context. A context-free morphological check cannot
distinguish a correct word from an incorrect one if both are morphologically
valid, which is exactly the situation on this dataset. More powerful
morphological tools, such as context-aware disambiguation, or a different
intervention strategy (suggest rather than revert) might activate on more cases.

\subsection{Few-Shot as the Least Harmful Augmentation}

Among the augmented conditions, few-shot (Phase~4B) causes the smallest
degradation. This is consistent with the general finding that demonstrations
constrain the output distribution more precisely than rule descriptions or
retrieved passages. The QALB examples, despite originating from typing rather
than OCR, share enough surface characteristics with OCR errors that they
partially redirect the model's corrections toward genuine error patterns. Whether
using OCR-native examples---pairs generated by running an OCR engine on clean text
and pairing with the original---would push few-shot performance above zero-shot
is an open question worth exploring.

\subsection{Limitations}

This study evaluates one dataset, one OCR engine, and one LLM. The PATS-A01
Akhbar font has a low baseline CER on normal samples (1.37\%), which sets a
very low bar that the LLM cannot clear. A different OCR engine with higher
baseline errors, or a handwritten dataset like KHATT, might show a different
balance. It is entirely possible that knowledge augmentation is beneficial
when the OCR output is sufficiently degraded---the model then has more to
correct and less risk of over-correcting accurate tokens.

The 200-sample evaluation is sufficient for the comparison purposes of this
study but would need to expand for publication-ready statistical claims.

% -------------------------------------------------------------------------
\section{Conclusion}
% -------------------------------------------------------------------------

We tested six prompt-engineering strategies for Arabic post-OCR correction using
a 4-billion-parameter instruction-tuned LLM. The results were clear: zero-shot
correction with a conservative instruction is the best-performing LLM strategy
on high-quality typewritten Arabic OCR output, and every form of additional
context we tried made things worse. Confusion-matrix injection and RAG caused
the most damage; few-shot examples caused the least. Morphological post-processing
had no effect.

The practical implication is that deploying a more complex system does not
automatically improve results---and can actively harm them if the baseline OCR
is already reasonably accurate. The most valuable contribution of the LLM in
this experiment is handling the Qaari repetition artifact, a failure mode that
no post-processing heuristic would easily catch.

Future work should evaluate these strategies on handwritten Arabic (e.g.,
KHATT), where OCR errors are more frequent and diverse, and test whether the
degradation pattern reverses when the baseline CER is substantially higher.
Fine-tuned models on OCR-specific correction data are another direction, since
instruction tuning on genuine OCR error--correction pairs would teach the model
when \textit{not} to intervene, which is the lesson that prompt engineering alone
seems unable to convey.
